Тест-кейсы для проверки бизнес-логики веб-приложения https://www.sixt.com


Тест-кейс № 1
"Авторизация пользователя при корректном заполнении формы"
	1. Пользователь открывает форму авторизации посредством нажатия на главной странице кнопки "Log in"
	2. В поле "Email" пользователь вводит ранее зарегистрированный корректный адрес электронной почты
	3. Пользователь нажимает на кнопку "Continue"
	4. В поле "Password" пользователь вводит корректный пароль
	5. Пользователь нажимает на кнопку "Log in"
Результат: Происходит вход в аккаунт и перенаправление на главную страницу с инициалами профиля в правом верхнем углу 

Тест-кейс № 2
"Ввод в форму авторизации ранее незарегистрированного адреса электронной почты"
	1. Пользователь открывает форму авторизации посредством нажатия на главной странице кнопки "Log in"
	2. В поле "Email" формы авторизации пользователь вводит ранее незарегистрированный в системе адрес электронной почты
	3. Пользователь нажимает кнопку "Continue"
Результат: Происходит перенаправление на страницу формы регистрации в системе

Тест-кейс № 3
"Выбор локации, в которой не существует арендной станции"
	1. На главной странице в форме заполнения аренды пользователь выбирает поле "Find a location"
	2. Пользователь вводит "Gabon"
Результат: Выдается сообщение "Sorry, but there are no SIXT stations available near Gabon!"

Тест-кейс № 4
"Дата окончания аренды раньше, чем дата начала аренды"
Предусловие: Текущая дата - JAN 02, текущее время - 11:00 AM
	1. На главной странице в форме заполнения аренды пользователь выбирает поле "Pick-up date"
	2. Пользователь выбирает значение JAN 07
	3. Пользователь выбирает поле "Pick-up time"
	4. Пользователь выбирает значение 04:30 PM
	5. Пользователь выбирает поле "Return date"
	6. Пользователь выбирает значение JAN 06
Результат: Поле "Pick-up date" автоматически меняет значение на JAN 05
